# PolyCopilot — 项目说明与报告解读

> 供团队快速理解项目架构和 JSON 报告字段。最后更新：2026-03

---

## 一、项目概述

**输入钱包地址 → 输出跟单指数 (0-100) 及等级 (S/A/B/C/D)**，判断该 Polymarket 玩家是否值得跟单。  
核心价值：量化胜率/盈亏比/入场时机、识别高频/做市商、评估仓位管理能力、估算跟单起步资金。

---

## 二、架构与运行

**数据流**：Fetcher（API 拉取）→ Processor（数据清洗、PnL 对齐）→ Analyzer（分析、评分、报告）

| 模块 | 文件 | 职责 |
|------|------|------|
| Fetcher | `fetcher.py` | 拉取 activity、closed_positions、positions、市场元数据；高频预检 |
| Processor | `processor.py` | 以 (conditionId, outcome) 对齐 PnL，关联 Gamma 元数据 |
| Analyzer | `analyzer.py` | Event 聚合、行为分析、跟单评分、JSON 报告 |

**运行**：`python tests/test_full_analysis.py`，修改文件顶部 `ADDR` 可切换地址。  
**输出**：终端打印 + `data/{地址前10位}_report.json`

**依赖**：`pip install loguru tenacity httpx pandas pyarrow numpy`

## 三、报告字段速查

| 字段 | 功能 |
|------|------|
| `summary` | 总览：PnL、胜率、ROI、活跃期（Outcome 级 + Event 级） |
| `entry_timing` | 入场优势：entry_edge = 结算价 - 均价，越高越好 |
| `category_breakdown` | 按分类统计利润占比，看 Alpha 集中在哪 |
| `pressure_test` | 三套情景下的峰值资金占用、建议起步资金 |
| `behavior` | 交易习惯：持仓时长、持仓风险(label: scalper/market_maker/normal)、交易频率、仓位分布、PnL 集中度 |
| `conviction_analysis` | 仓位管理：重仓 vs 轻仓胜率差、follow_threshold |
| `event_trade_summary` | **每个事件的买/卖次数、VWAP、价格离散、执行路径标签** |
| `events` | 每个事件的 PnL、ROI、投入、entry edge（财务维） |
| `copy_trading_score` | 总分、等级、各维度拆解、风险提示 |
| `validation` | 8 项数据校验 + 字段假设审计 |

**关键区分**：`event_trade_summary` = 行为（怎么买的）；`events` = 财务（赚了多少）。下单次数在 `event_trade_summary.buy_count` / `sell_count`，用 `event_slug` 关联。

---

## 四、快速判断

- **是否适合跟单**：看 `copy_trading_score.total`、`grade`、`risk_warnings`
- **是否回避**：`behavior.holding_risk.label` 为 `scalper` 或 `market_maker_suspect` 时不宜跟单
- **每事件下单次数**：`event_trade_summary[].buy_count`、`sell_count`
