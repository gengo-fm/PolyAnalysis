## Why

当前 Polymarket 聪明钱分析系统存在严重效率问题：
1. **重复获取数据**：每次运行都从 API 重新拉取，没有利用缓存
2. **全量更新**：即使只有少量新数据，也拉取全部历史记录
3. **不可续传**：中断后无法从断点继续，浪费 API 调用
4. **速度慢**：每个钱包平均 30 秒，1300+ 钱包需要 10+ 小时

通过实现增量数据获取和智能缓存，可以将效率提升 10-20 倍，同时确保数据不重复获取、支持断点续传。

## What Changes

### 核心改动

1. **三层缓存架构**
   - L1: 内存缓存（当前会话）
   - L2: 文件缓存（持久化 JSON）
   - L3: API（首次全量，后续增量）

2. **智能增量更新**
   - 基于时间戳的增量拉取
   - 数据指纹检测变化
   - 自动合并新旧数据

3. **断点续传机制**
   - 分析进度持久化
   - 失败重试队列
   - 支持中断恢复

4. **批量并发处理**
   - 20 个并发请求
   - 智能速率限制
   - 失败自动跳过

5. **元数据管理**
   - 数据版本控制
   - 数据指纹
   - 更新时间追踪

## Capabilities

### New Capabilities

- `incremental-fetch`: 基于时间戳的增量数据获取，避免重复拉取
- `smart-cache`: 三层缓存架构，智能选择数据来源
- `progress-tracking`: 分析进度持久化，支持断点续传
- `batch-processing`: 批量并发处理，提升整体效率

### Modified Capabilities

- `data-fetcher`: 修改为支持增量模式，原有全量拉取作为后备

## Impact

### 受影响代码

- `polycopilot/fetcher.py` — 核心数据获取逻辑
- `polycopilot/cache.py` — 缓存管理
- `analyze.py` — CLI 入口，增加批量模式

### 受影响系统

- Polymarket API 调用（减少 90% 以上重复请求）
- 文件系统存储（增加元数据管理）
- 用户体验（分析速度大幅提升）

### 预期效果

- **首次运行**：完整分析 1300+ 钱包，约 2-3 小时
- **后续运行**：增量更新，仅处理新数据，分钟级完成
- **API 调用**：减少 90%+
- **可靠性**：支持中断恢复，不丢失进度
